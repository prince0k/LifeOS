## 1. Purpose
The purpose of this document is to detail the single-user persona for whom **LifeOS** is built. By keeping the design highly focused on one persona, we prevent feature bloat and ensure every module solves concrete, real-life difficulties.

---

## 2. Objectives
- Define the specific constraints, schedule structures, and pain points of the target user.
- Serve as a design filter for prioritizing UI layouts, quick logs, and notification timings.
- Establish a user baseline for testing the adaptive timetable recommendations.

---

## 3. Scope
This document covers the personal profile, rotating shift schedule characteristics, cognitive loads, and feature requirements of the single target user of LifeOS.

---

## 4. Requirements

| Requirement ID | Description | Priority | Traceability |
|---|---|---|---|
| **REQ-PERS-001** | The UI layout must adapt visually to accommodate shift-specific priorities (e.g., Night Shift puts "Mailing Tasks" first, Morning Shift emphasizes evening "Deep Work"). | High | MOD-Dashboard |
| **REQ-PERS-002** | The app must allow one-tap switches between shift templates on the fly. | Critical | MOD-Dashboard |
| **REQ-PERS-003** | Alarm clocks and local reminder systems must adapt to shift timings. | Critical | MOD-Settings |

### 4.1 Persona Profile: The Rotating Shift Builder

- **Age/Background:** Developer and administrator managing multiple local and remote projects (Mailing, CityHost).
- **Work Schedule:** High variability, rotating across four core states:
  - **Morning Shifts (10:30 AM – 6:30 PM):** Concentrated work time, leaving evenings for deep project work.
  - **Night Shifts (7:30 PM – 3:30 AM):** Highly disruptive to biological sleep cycles. Tasks must shift to the afternoon before work.
  - **12-Hour Shifts (12:00 PM – 12:00 AM):** Extremely exhausting. Requires minimum cognitive load and maximum recovery focus.
  - **Off Days:** Unrestricted productivity time. Long deep focus work, planning, and recreation.

### 4.2 Pain Points & Frustrations
1. **Biological Clock Disruption:** Difficulty maintaining regular sleep times, leading to fatigue.
2. **Cognitive Load:** Forgetting what tasks to work on next because the schedule is constantly changing.
3. **Productivity Guilt:** Feeling like a failure when missing tasks on a 12-hour shift day, when recovery should actually be prioritized.
4. **Friction in Trackers:** Traditional habits/notes trackers require too many clicks or setup, leading to logging abandonment.

## 6. Edge Cases
- **Unplanned Shifts / Shift Overlaps:** If a user works double-shifts, the app must not crash. It must fallback to the *12-Hour Shift* (Recovery-focused) preset to prevent exhaustion.
- **Varying Energy Levels within a Shift:** Even on an *Off Day*, the user might have very low recovery. The system must prioritize the *Recovery Score* over the *Shift Template* when recommending task volumes.
